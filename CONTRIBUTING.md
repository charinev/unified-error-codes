# CharIN Unified Error Codes Contribution Guide

This document outlines the contribution process for the CharIN Unified Error Codes specification, a joint effort of CharIN Members and external parties.

Contributions are made via GitHub Issues and Pull Requests. All submissions will be reviewed regularly by the project maintainers.

## Issues

All new proposals, changes, or identified problems should be filed as a GitHub Issue.

Before creating a new issue, please search the existing issues to ensure the topic has not already been raised. New issues must provide sufficient technical detail and, if proposing a change, a clear proposal to establish a basis for discussion.

## Pull Requests

Modifications to the specification must be submitted as a Pull Request (PR) against the `main` branch.

* All PRs will be reviewed and commented on by the maintainers.
* Once a PR is approved by the maintainers and any merge conflicts are resolved, it will be merged.
* Merged PRs will be incorporated into the official specification draft.
